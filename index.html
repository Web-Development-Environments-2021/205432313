web version אפשרויות כתיבה\קריאה מנועי חיפוש סטטיות ובינה מלאכותית
web 1.0 חברות ועיתונאים מפרסמות תוכן שנצרך על ידי האוכלוסייה הכללית(אתרי חדשות למשל) In Web 1.0 search engines retrieve macro contents. Search is very fast but many times results are inaccurate or more than users can chew. Web 1.0 was all about static content, one way publishing of content without any real interaction between readers or publishers or each other.
web 2.0 . אפשרות לכל אדם לפרסם תוכן דרך פלטפורמות מתאימות כמו: ויקיפדיה, יוטיוב ועוד. אנשים צורכים תוכן המופק על ידי אנשים פרטיים In Web 2.0 search engines retrieve tags with micro contents (Furl even retrieves tags with macro contents). The process of tagging is manual, tedious and covers negligible percents of the WWW. Web 2.0 tags every- thing: pictures, links, events, news, Blogs, audio, video, and so on. Google Base even retrieves micro content texts. Web 2.0 is more about 2 way communication through social networking, blogging, wikis, tagging, user generated content and video.
web 3.0 חברות בנו פלטפורמות ש"מתקשרות" עם כל אדם, ומעצבות את חווית הגלישה שלו לפי למידת מכונה. זה גם אפשר לחברות לפרסם לכל אדם תכנים הרלוונטים לו(פייסבוק ועוד) In Web 3.0 search engines will hopefully retrieve micro content texts which were tagged automatically. This implies translating billions of Web 1.0 macro contents into micro contents. The result could be more precise search because tagging can solve part of the ambiguity that homonyms and synonyms introduce into the process of Search. Web 3.0 is curiusly undefined. AI and the web learning what you want and delivering you a Personalized web experience.
Use case example - most advanced cell phone
- web 1.0: we must write a specific url in order to reach the data.
- web 2.0: using "most advanced cell phone" as a query to a search engines and we will see result from the query as web tags.
- web 3.0: using "most advanced cell phone" as a query to a search engines and we will see result that contain the query words and semantic words as tags or web info.
